# 밸런스메이커 UI · 기획 메모

목업(`mockup/pages/main-clan.html`)의 **맵·영웅 밴(편집 단계)** 안내는 **이 문서에만** 둔다. 해당 구간에는 `?` 툴팁을 두지 않는다. 그 외 화면(프리미엄 줄, 참고 패널, 단계 탭 등)의 짧은 안내는 목업 `data-tip`과 아래 절을 맞춘다.

PRD 상세는 `PRD.md` §6.1 등과 함께 본다.

---

## 페이지 개요

- 운영진이 **출전 명단·포지션**을 잡는 편집기.
- **5v5 · 탱1 딜2 힐2** 고정.
- **맵**은 맵 밴이 꺼져 있을 때만 직접 선택(모달). 맵 밴이 켜지면 최종 맵은 **② 맵 밴픽 세션**에서 결정(아래 스펙).
- 상단 **단계 탭**(편집 / 밴픽 / 경기 진행)으로 배치 확정 이후 화면을 전환해 미리 본다. (관전·승부예측은 ③에 통합, 목업 참고)

---

## 세션 플로우 (밸런스장 · 구성원)

절차의 **중심은 웹(ClanSync)** 이다. 디스코드 연동은 **선택적 보조**(알림 등)이며, 밴픽·예측의 필수 경로가 아니다.

### 역할

| 구분 | 내용 |
|------|------|
| **밸런스장** | 밸런스 메이커로 **세션을 연 운영진 1명**. **① 편집(배치)** 화면만 사용한다. |
| **그 외 운영진** | 밸런스장이 아니면 **구성원과 동일한 화면**(편집 권한 없음). |
| **구성원** | 세션에 참가해 경기 진행·밴픽·승부예측 등을 본다(참가 여부에 따라 분기). 세션 미시작 시 **대기 화면**만 표시, 밸런스장이 세션을 열면 실시간 합류. 목업은 `mockup/pages/main-clan.html` + `mockApplyBalanceSessionGate`(`clan-mock.js`), `?balanceSession=1` 또는 sessionStorage로 라이브 UI 전환. |

### 단계 요약

1. **세션 생성** — 밸런스장이 밸런스 메이커를 눌러 세션 생성.
2. **배치(편집)** — 밸런스장은 편집 UI. 나머지(다른 운영진 포함)는 **경기 진행(관전) 형태**로 동일 보드를 보며, 맵·플레이어 변경이 **실시간** 반영된다.
3. **배치 완료** — 밸런스장이 **배치 완료**를 누른다.
4. **밴픽** — 맵/영웅 밴이 **켜진 시나리오**에서는, 배치 완료 직후 **밸런스장·참가자는 밴픽 화면으로 리디렉션**되어 기존 밴 규칙대로 진행한다. **비참가 구성원**도 밴픽이 진행 중이면 **실시간으로 밴픽 화면을 관전**할 수 있다(동일 세션 동기화).
5. **경기 시작(정의)** — **경기 시작 = 맵·영웅 등 해당 시나리오의 밴이 모두 끝난 직후.** 밴이 없는 시나리오에서는 **배치 완료 직후**를 경기 시작으로 본다.
6. **승부예측 마감 타이머** — **경기 시작 시점**에 시작한다. 밸런스장(운영진) 화면에는 경기 시작과 함께 **결과 입력·승리 팀 선택·승부예측** 등이 노출되고, 구성원(비출전) 화면에는 **승부예측**이 노출된다.
7. **경기 종료** — 밸런스장이 **결과 입력**으로 승리 팀을 확정한다. 비출전자는 예측이 유효한 경우 **클랜코인 등 배당 정산**.
8. **예측 무효** — **동점·무효·재경기**로 판정된 경기는 해당 판의 **승부예측을 무효** 처리한다.
9. **사후 정정** — 오입력·누락·추가 반영은 **클랜 관리의 내전 히스토리(예정 탭)** 등에서 처리한다. 실시간 플로우의 “백업” 역할은 여기에 둔다.

## 편집 카드 · 배치 확정

- 목업은 **시나리오 미리보기**다. 실제 구현에서는 배치 완료 시 참가·밴픽 옵션·역할에 따라 **② 밴픽 / ③ 경기 진행**(승부예측·결과 입력 포함)으로 분기·리디렉트한다.

## 맵 밴 · 영웅 밴 (편집 ① — 문서 전용 카피)

아래는 목업·구현 시 **단일 소스**로 쓴다.

### 편집 ① — 맵 밴 OFF

- **맵**은 모달에서 직접 선택.

### 편집 ① — 맵 밴 ON

- **맵 라벨·맵 선택 버튼·모드 문구**는 표시하지 않는다 (`display: none` 수준). 맵 선택 예외 경로는 *「맵 밴이 켜져 있어 이 단계에서는 맵 선택이 없습니다. ② 맵 밴픽 세션에서 맵이 정해집니다.」* (목업 `alert`).
- **배치 완료 버튼 위**에 **맵 풀 분류 드롭다운**을 둔다: 전체, 쟁탈, 호위, 밀기, 혼합, 플래시 포인트 등(서버·DB의 맵 분류와 일치).
- 편집 단계에는 **밴 슬롯 UI를 두지 않는다.**

### ② 맵 밴픽 세션 (참가자 · 관전)

1. **배치 완료 시점**에 참가자·밸런스장은 **웹 세션**의 밴픽 화면으로 진입한다(리디렉션). **Premium·디스코드 연동**이 켜인 경우에는 **선택적으로** 동일 시점에 디스코드 알림(링크)을 보낼 수 있으나, **필수 경로는 아니다**.
2. 세션 **제한 시간 15초**, 남은 시간 **실시간 표시**.
3. 앞서 선택한 **분류에 해당하는 맵 풀**에서 **3개를 무작위**로 뽑아 표시한다.
4. **투표 규칙(A안)**: 경기 **참가자**는 후보 **3개 중 1개에만 1표**. 미투표는 별도 페널티 없음 — **각 맵 득표수만** 가중치로 쓴다.
5. **15초 종료 후** 세 맵의 득표수를 **가중치**로 하여 **맵 1개를 무작위 확정**한다.
6. **동률**: 득표가 **완전 동일**한 경우(예: 3·3·3 또는 0·0·0) 가중치를 **1:1:1**로 보고 **균등 1/3**으로 확정한다.
7. **확정 연출(참고)**: 오버워치 2 맵 투표는 세 후보를 순환하는 **룰렛(roll-through)** 이 멈추며, 득표가 많은 맵에 더 오래 머무는 식으로 보인다. 웹 목업에서는 **동일한 가중 확정 결과**에 맞춰 룰렛·감속·최종 하이라이트만 재현한다(게임과 1:1 동일한 타이밍·에셋은 아님).

### ③ 영웅 밴픽 세션 (참가자, OW2 경쟁전 참고)

**순서**: **맵 밴픽이 끝난 직후** 같은 밴픽 흐름에서 이어진다. 맵 밴 OFF·영웅 밴 ON만 켠 경우에는 **배치 완료 직후** 영웅 밴만 진행한다.

**OW2 요약(참고)**  
- 참가자는 **밴할 영웅 3명**을 **순위**로 고른다(동시 투표).  
- 가중: **1순위 7점, 2순위 5점, 3순위 3점**(공식 설명·가이드 기준).  
- 전체적으로 **최대 4명**이 밴에 올라가며, **역할당 최대 2명** 등 제한이 있다(역할 초과 시 다음 순위로 밀림).  
- 클랜 내전 목업에서는 **가중 합산 → 점수 순으로 최대 4명 선택, 역할당 최대 2명**으로 단순화한다.

**클랜Sync 목업 규칙**

1. 세션 **제한 시간 20초**(목업), 남은 시간 실시간 표시.  
2. 참가자는 서로 다른 영웅 **3명을 순서대로 선택**(1·2·3순위). 선택마다 **7 / 5 / 3**을 해당 영웅 누적 점수에 더한다(여러 명이 투표하면 합산).  
3. **「투표 반영」** 으로 한 참가자 분을 합산에 반영하고 선택을 비운다(목업). **「랜덤 투표 시뮬」** 로 다수 참가를 흉내 낸다.  
4. 시간 종료 시: 아직 반영 안 된 **현재 3명 선택**이 있으면 자동 반영.  
5. **확정**: 누적 점수 내림차순으로, **역할당 최대 2명** 제한을 지키며 **최대 4명**을 밴 목록으로 확정.  
6. 영웅 밴 종료 후 **③ 경기 진행**(라인업)으로 이동하고, **밴 목록**을 한 줄로 표시한다.

### 단계별 화면 (워크플로 탭)

| 탭 | 설명 |
|----|------|
| ② 밴픽 | **맵 밴픽**(맵 ON) → **영웅 밴픽**(영웅 ON) 순. 둘 다 OFF면 이 단계 생략. 비참가자도 **실시간 밴픽 관전** 가능. |
| ③ 경기 진행 | 라인업·승부예측·결과 입력. **경기 시작(밴 종료 직후)** 이후 예측 마감 타이머 시작. 목업에서 ④ 관전은 ③에 통합. |

## 집계일 (KST) — 도메인 규칙

밸런스 편집 목업에는 **집계일 입력 UI를 두지 않는다.** 아래는 API·아카이브 귀속용 규칙만 정리한다.

- **당일 정의**: 해당 일자의 **06:00**(KST)부터 **다음 날 06:00** 직전까지를 같은 집계일로 본다. 자정을 넘긴 새벽 경기도 같은 집계일에 묶인다.
- **표시 일자·아카이브 귀속**은 백엔드/API에서 처리하고, 클라이언트는 응답값만 사용한다.

## 목업 허브 · 플랜

- `_hub.html`에서 **Free / Premium** 전환 시 Premium 전용 UI(예: 승부 예측 블록) 노출이 달라진다.
- Free 플랜 목업에서 해당 UI는 숨기고, **도움말**로 Premium에서 확인하도록 안내한다.
- `_hub.html` 권한·플랜 라벨 옆 **도움말(?)** — 툴팁은 fixed 레이어로 표시(사이드바 스크롤과 무관).

## 도움말 툴팁 본문 (목업 `data-tip`과 동기)

**맵·영웅 밴(편집 행)** 은 위 「맵 밴 · 영웅 밴」절만 참고한다. 나머지 화면의 `?`는 구현 시 이 절과 맞추거나 i18n 키로 분리한다.

### Premium 기능 줄 (편집 상단)

- **팀 자동 밸런싱**: Premium 클랜 기능. 자동 점수·포지션(탱·딜·힐)을 반영해 블루·레드 진영을 자동 분배. 운영진은 결과를 수동 조정 가능.
- **OCR 닉 입력**: Premium. 내전·스크림 결과 화면 스크린샷에서 닉네임 인식으로 명단 입력 보조.
- **디스코드 시작 알림(자동, 선택)**: Premium. 연동 시 배치 완료 시점에 디스코드로 시작 안내·밴픽 URL을 보낼 수 있으나, **웹 세션이 본 경로**이며 디스코드는 보조다.

### 승률 예측(참고 패널)

- Premium 참고 지표. 팀 자동점수 합 기반 승률 추정(목업). 실제 배당·예측 마감은 정책 따름.

### 경기 진행 · 승부 예측 (③, 목업에서 관전 탭 통합)

- Premium. 비출전 구성원이 블루/레드 승 예측·클랜코인. **예측 마감 타이머는 경기 시작(밴 종료 직후)** 에 맞춘다. 마감·배당·파리뮤추엘 정산은 서버 정책.

### 내전 히스토리(예정)

- 클랜 관리에서 **오입력 수정·사후 요소 추가** 등. 실시간 플로우와 별도.

### 기타

- 단계 탭·밴픽·경기 진행 등 나머지 `?` 본문은 목업 `data-tip`과 동기 유지한다.
