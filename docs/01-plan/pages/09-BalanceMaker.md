# 09 BalanceMaker UI · 기획 메모

> 기존 `docs/01-plan/balance-maker-ui-notes.md` 내용을 이 문서로 통합했다.

목업(`mockup/pages/main-clan.html`)의 **맵·영웅 밴(편집 단계)** 안내는 **이 문서에만** 둔다. 해당 구간에는 `?` 툴팁을 두지 않는다. 그 외 화면(프리미엄 줄, 참고 패널, 단계 탭 등)의 짧은 안내는 목업 `data-tip`과 아래 절을 맞춘다.

PRD 상세는 `PRD.md` §6.1 등과 함께 본다.

## 페이지 개요

- 운영진이 **출전 명단·포지션**을 잡는 편집기.
- **5v5 · 탱1 딜2 힐2** 고정.
- **맵**은 맵 밴이 꺼져 있을 때만 직접 선택(모달). 맵 밴이 켜지면 최종 맵은 **② 맵 밴픽 세션**에서 결정(아래 스펙).
- 상단 **워크플로 단계 탭**으로 배치 확정 이후 화면을 전환해 미리 본다. **목업 라벨**: `① 편집 중` → `② 참가자 · 맵 밴픽` → `③ 경기 진행`. **④ 관전** 탭은 DOM에만 두고 숨기며, 승부예측·예측 표는 **③에 통합**(문서·목업 동일).

### 참가자 네임플레이트 (경기 진행 화면)

- **경기 진행(관전) 단계**에서 블루/레드 슬롯에 각 플레이어가 **장착한 네임플레이트**(배경·프레임·엠블럼·칭호 등)를 표시한다.
- 에셋은 **서비스에서 제공하는 프리셋만** 사용한다. **사용자 이미지 업로드**로 네임플레이트를 꾸미는 기능은 **제공하지 않는다**(PRD 「꾸미기 에셋 정책」).
- **네임카드에 붙는 업적 뱃지**는 프로필에서 **등록한 게임마다** 표시 목록을 따로 두고, **최대 5개**(앞쪽 우선)로 둔다. 정적 목업: `mockup/scripts/app.js`의 `MOCK_BADGE_NAMEPLATE_MAX`·`mockBadgeCaseGetPicks()`(게임 키 `ow` / `val`), `profile.html`·`badge-case-modal.html`.
- 해금: 클랜 스토어(클랜코인), 이벤트·월간 미니게임 등 보상(기획).

## 세션 플로우 (밸런스장 · 구성원)

절차의 **중심은 웹(ClanSync)** 이다. 디스코드 연동은 **선택적 보조**(알림 등)이며, 밴픽·예측의 필수 경로가 아니다.

### 역할

| 구분 | 내용 |
|------|------|
| **밸런스장** | 밸런스 메이커로 **세션을 연 운영진 1명**. **① 편집(배치)** 화면만 사용한다. |
| **그 외 운영진** | 밸런스장이 아니면 **구성원과 동일한 화면**(편집 권한 없음). |
| **구성원** | 세션에 참가해 경기 진행·밴픽·승부예측 등을 본다(참가 여부에 따라 분기). 세션 미시작 시 **대기 화면**만 표시, 밸런스장이 세션을 열면 실시간 합류. 목업은 `mockup/pages/main-clan.html` + `mockApplyBalanceSessionGate`(`clan-mock.js`), `?balanceSession=1` 또는 sessionStorage로 라이브 UI 전환. |

### 단계 요약

1. **세션 생성** — 밸런스장이 밸런스 메이커를 눌러 세션 생성.
2. **배치(편집)** — 밸런스장은 편집 UI. 나머지(다른 운영진 포함)는 **경기 진행(관전) 형태**로 동일 보드를 보며, 맵·플레이어 변경이 **실시간** 반영된다.
3. **배치 완료** — 밸런스장이 **배치 완료**를 누른다.
4. **밴픽** — 맵/영웅 밴이 **켜진 시나리오**에서는, 배치 완료 직후 **밸런스장·참가자는 밴픽 화면으로 리디렉션**되어 기존 밴 규칙대로 진행한다. **비참가 구성원**도 밴픽이 진행 중이면 **실시간으로 밴픽 화면을 관전**할 수 있다(동일 세션 동기화).
5. **경기 시작(정의)** — **경기 시작 = 맵·영웅 등 해당 시나리오의 밴이 모두 끝난 직후.** 밴이 없는 시나리오에서는 **배치 완료 직후**를 경기 시작으로 본다.
6. **승부예측 마감 타이머** — **경기 시작 시점**에 시작한다. 밸런스장(운영진) 화면에는 경기 시작과 함께 **결과 입력·승리 팀 선택·승부예측** 등이 노출되고, 구성원(비출전) 화면에는 **승부예측**이 노출된다.
7. **경기 종료** — 밸런스장이 **결과 입력**으로 승리 팀을 확정한다. 비출전자는 예측이 유효한 경우 **클랜코인 등 배당 정산**. 확정 순간 **승자·내 적중 여부·배당 코인**을 담은 **결과 팝업 1회**를 대상자에게 노출한다("경기 종료 · 결과 팝업" 섹션 참조). 별도 "매치 결과 카드" 컴포넌트는 두지 않는다.
8. **예측 무효** — **동점·무효·재경기**로 판정된 경기는 해당 판의 **승부예측을 무효** 처리한다.
9. **사후 정정** — 오입력·누락·추가 반영은 **클랜 관리의 내전 히스토리(예정 탭)** 등에서 처리한다. 실시간 플로우의 “백업” 역할은 여기에 둔다.

- 목업은 **시나리오 미리보기**다. 실제 구현에서는 배치 완료 시 참가·밴픽 옵션·역할에 따라 **② 밴픽 / ③ 경기 진행**(승부예측·결과 입력 포함)으로 분기·리디렉트한다.

## 맵 밴 · 영웅 밴 (편집 ① — 문서 전용 카피)

아래는 목업·구현 시 **단일 소스**로 쓴다.

### 편집 ① — 맵 밴 OFF

- **맵**은 모달에서 직접 선택.

### 편집 ① — 맵 밴 ON

- **맵 라벨·맵 선택 버튼·모드 문구**는 표시하지 않는다 (`display: none` 수준). 맵 선택 예외 경로는 *「맵 밴이 켜져 있어 이 단계에서는 맵 선택이 없습니다. ② 맵 밴픽 세션에서 맵이 정해집니다.」* (목업 `alert`).
- **배치 완료 버튼 위**에 **맵 풀 분류 드롭다운**을 둔다: 전체, 쟁탈, 호위, 밀기, 혼합, 플래시 포인트 등(서버·DB의 맵 분류와 일치).
- 편집 단계에는 **밴 슬롯 UI를 두지 않는다.**

### ② 맵 밴픽 세션 (참가자 · 관전)

1. **배치 완료 시점**에 참가자·밸런스장은 **웹 세션**의 밴픽 화면으로 진입한다(리디렉션). **Premium·디스코드 연동**이 켜인 경우에는 **선택적으로** 동일 시점에 디스코드 알림(링크)을 보낼 수 있으나, **필수 경로는 아니다**.
2. 세션 **제한 시간 15초**, 남은 시간 **실시간 표시**.
3. 앞서 선택한 **분류에 해당하는 맵 풀**에서 **3개를 무작위**로 뽑아 표시한다.
4. **투표 규칙(A안)**: 경기 **참가자**는 후보 **3개 중 1개에만 1표**. 미투표는 별도 페널티 없음 — **각 맵 득표수만** 가중치로 쓴다.
5. **15초 종료 후** 세 맵의 득표수를 **가중치**로 하여 **맵 1개를 무작위 확정**한다.
6. **동률**: 득표가 **완전 동일**한 경우(예: 3·3·3 또는 0·0·0) 가중치를 **1:1:1**로 보고 **균등 1/3**으로 확정한다.
7. **확정 연출(참고)**: 오버워치 맵 투표는 세 후보를 순환하는 **룰렛(roll-through)** 이 멈추며, 득표가 많은 맵에 더 오래 머무는 식으로 보인다. 웹 목업에서는 **동일한 가중 확정 결과**에 맞춰 룰렛·감속·최종 하이라이트만 재현한다(게임과 1:1 동일한 타이밍·에셋은 아님).

### ③ 영웅 밴픽 세션 (참가자, OW 경쟁전 참고)

**순서**: **맵 밴픽이 끝난 직후** 같은 밴픽 흐름에서 이어진다. 맵 밴 OFF·영웅 밴 ON만 켠 경우에는 **배치 완료 직후** 영웅 밴만 진행한다.

**OW 요약(참고)**  
- 참가자는 **밴할 영웅 3명**을 **순위**로 고른다(동시 투표).  
- 가중: **1순위 7점, 2순위 5점, 3순위 3점**(공식 설명·가이드 기준).  
- 전체적으로 **최대 4명**이 밴에 올라가며, **역할당 최대 2명** 등 제한이 있다(역할 초과 시 다음 순위로 밀림).  
- 클랜 내전 목업에서는 **가중 합산 → 점수 순으로 최대 4명 선택, 역할당 최대 2명**으로 단순화한다.

**클랜Sync 목업 규칙**

1. 세션 **제한 시간 20초**(목업), 남은 시간 실시간 표시.  
2. 참가자는 서로 다른 영웅 **3명을 순서대로 선택**(1·2·3순위). 선택마다 **7 / 5 / 3**을 해당 영웅 누적 점수에 더한다(여러 명이 투표하면 합산).  
3. **「투표 반영」** 으로 한 참가자 분을 합산에 반영하고 선택을 비운다(목업). **「랜덤 투표 시뮬」** 로 다수 참가를 흉내 낸다.  
4. 시간 종료 시: 아직 반영 안 된 **현재 3명 선택**이 있으면 자동 반영.  
5. **확정**: 누적 점수 내림차순으로, **역할당 최대 2명** 제한을 지키며 **최대 4명**을 밴 목록으로 확정.  
6. 영웅 밴 종료 후 **③ 경기 진행**(라인업)으로 이동하고, **밴 목록**을 한 줄로 표시한다.

### 단계별 화면 (워크플로 탭)

| 목업 탭 라벨 | 설명 |
|----|------|
| ① 편집 중 | 배치 편집(밸런스장) + 참고 패널·배치 완료. |
| ② 참가자 · 맵 밴픽 | **맵 밴픽**(맵 ON) → **영웅 밴픽**(영웅 ON) 순. 둘 다 OFF면 이 단계 생략. 비참가자도 **실시간 밴픽 관전** 가능. |
| ③ 경기 진행 | 라인업·**비출전 승부예측**·결과 입력. **경기 시작(밴 종료 직후)** 이후 예측 마감 타이머 시작. |
| ④ (통합됨) | 목업에서 숨김. 승부예측 UI는 ③ 우측 사이드바(`mock-hide-on-free`). |

## 집계일 (KST) — 도메인 규칙

밸런스 편집 목업에는 **집계일 입력 UI를 두지 않는다.** 아래는 API·아카이브 귀속용 규칙만 정리한다.

- **당일 정의**: 해당 일자의 **06:00**(KST)부터 **다음 날 06:00** 직전까지를 같은 집계일로 본다. 자정을 넘긴 새벽 경기도 같은 집계일에 묶인다.
- **표시 일자·아카이브 귀속**은 백엔드/API에서 처리하고, 클라이언트는 응답값만 사용한다.

## 참고 패널

- 표시 순서: **점수 토글(Premium)** → **점수 차이 바** → **자동 점수 기반 승률 예측(Premium)**.
- **자동 점수·승률 예측** 등은 배치 판단 **보조**용이다.
- 실제 수치·노출 여부는 **API·구독(플랜) 정책**을 따른다.
- 점수 막대 제목은 **점수 토글 상태와 동기화**한다: `M 점수` 모드면 `M점수 차이`, `A 점수` 모드면 `A점수 차이`.
- 점수 차이/승률 예측 **막대 내부 수치 라벨은 팀명(블루/레드) 없이 숫자만** 노출한다(좌측=블루, 우측=레드).

## 목업 허브 · 플랜

- `_hub.html`에서 **Free / Premium** 버튼은 `sessionStorage`(`clansync-hub-mock-plan`)에 저장하고, 클랜 메인 iframe은 `pages/main-clan.html?role=…&plan=…`로 로드한다.

- Free 플랜 목업에서는 해당 UI를 숨기고, **③ 경기 진행** 하단 안내 + `data-tip`으로 허브 전환을 안내한다.
- `_hub.html` 권한·플랜 라벨 옆 **도움말(?)** — 툴팁은 fixed 레이어로 표시(사이드바 스크롤과 무관).

## 도움말 툴팁 본문 (목업 `data-tip`과 동기)

**맵·영웅 밴(편집 행)** 은 위 「맵 밴 · 영웅 밴」절만 참고한다. 나머지 화면의 `?`는 구현 시 이 절과 맞추거나 i18n 키로 분리한다.

### Premium 기능 줄 (편집 상단)

목업 `mock-premium-plan-tag`의 `data-tip` 원문과 동일하게 유지한다(한 줄씩 줄바꿈).

```
Premium 전용
• 자동 팀 밸런스
• OCR 닉 입력
• 디스코드 알림(선택, 웹이 본 경로)
```

- **자동 팀 밸런스**: 자동 점수·포지션(탱·딜·힐)을 반영해 진영 자동 분배. 운영진은 수동 조정 가능.
- **OCR 닉 입력**: 내전·스크림 결과 스크린샷에서 닉 인식으로 명단 입력 보조.
- **디스코드 알림(선택)**: 배치 완료 시점에 알림·링크 가능하나 **웹이 본 경로**, 디스코드는 보조.

### 승률 예측(참고 패널)

- Premium 참고 지표. 팀 자동점수 합 기반 승률 추정(목업). 실제 배당·예측 마감은 정책 따름.

### 경기 진행 · 승부 예측 (③, 목업에서 관전 탭 통합)

- Premium. 비출전 구성원이 블루/레드 승 예측·클랜코인. **예측 마감 타이머는 경기 시작(밴 종료 직후)** 에 맞춘다. 마감·배당·파리뮤추엘 정산은 서버 정책.

## 이름표(닉네임 슬롯) UI 상세

> 기존 `docs/01-plan/balance-nameplate-ui-plan.md` 내용을 이 절로 통합했다.

### 원칙

- **닉네임**은 슬롯 **좌측·상단**에 크게 둔다.
- **마이크 표시는 사용하지 않는다.**
- **오늘 전적**은 **승·무·패** 모두 표기한다. (집계일 KST 06:00 경계는 `schema.md`/`PRD.md`와 동일하게 서버가 내려주는 값을 표시한다고 가정.)
- **Manual 점수**: **Free·Premium 모두** 운영진이 편집 가능(목업은 `input`).
- **A**(자동 추정 값): 화면 라벨은 **`A`**만 쓰고 옆에 숫자를 둔다. **Premium에서만** 노출. Free에서는 **완전 비표시** (`body.mock-plan-free` + `mock-hide-on-free`).
- **추세 화살표/스파크라인**은 쓰지 않는다. 대신 자동 산정·규칙 엔진이 붙이는 **특이사항 태그(배지)**로 상태를 전달한다.

### 레이아웃 · 정보 우선순위

- **닉네임·오늘 전적**은 본문 **왼쪽**에 두고, **2줄 행** 오른쪽에는 **태그 레일 | M·A 점수 열**을 둔다.
- **M·A**는 **동일한 행 박스 스타일**(`.mock-balance-slot-score-row`), **세로로 쌓고 열 안에서는 우측 정렬**한다. **A는 M 바로 아래** 줄.
- **특이사항 태그**는 **M·A 열 바로 왼쪽**에 세로로 쌓는다(같은 `.mock-balance-slot-line2-tray` 안).
- 본문 2줄 안에서 좁아지면 아래 순으로 잘라낸다(마지막부터 숨김): 닉네임 → 오늘 전적 → `M` → `A`.

### 특이사항 태그

> **D-ECON-04** (DECIDED 2026-04-20) — 자동 산정 전용(수동 태깅 없음). Phase 1 카탈로그 13종. 본 클랜 내전(`match_type='intra'`)만 집계. [decisions.md §D-ECON-04](../decisions.md#d-econ-04--특이사항-태그-카탈로그).

- 규칙은 **서버/배치 스냅샷**에서 결정하고 UI는 배지만 담당한다. 사용자·운영자 수동 태깅은 허용하지 않는다.
- 스냅샷 저장소: `match_tags(user_id, clan_id, code, tone, map_id, computed_at, expires_at)` — 현재 유효 태그만 저장(이력 없음).

## 사이드바 알림 점 (`#sidebar-notify-balance` · D-SHELL-03)

- **트리거**: 내가 속한 **진행 중인 내전 세션**(`balance_sessions.status IN ('open','running')`) 수.
- **Clear**: `#balance` 뷰 진입 시 자동 clear. 세션이 아직 종료되지 않았다면 다음 refresh에 다시 점이 뜬다(정보성).
- **구성원·운영진 공통**: 세션 존재 자체가 시그널이므로 모든 역할에 동일하게 표시. 단, 구성원은 진입해도 "내전 시작" 버튼이 차단되므로 알림은 "관전할 거리 있음"의 의미로 기능.
- **매치 결과 기반 알림은 두지 않는다**: 경기 종료 결과 열람은 사이드바 알림이 아니라 **결과 입력 완료 시 1회 결과 팝업**으로 처리한다(아래 "경기 종료 · 결과 팝업" 섹션 참고). 별도 "매치 결과 카드" 컴포넌트를 만들지 않으므로 미확인 상태 자체가 정의되지 않고, 따라서 이 트리거는 구조적으로 존재하지 않는다.
- 자세한 규칙은 [decisions.md §D-SHELL-03](../decisions.md#d-shell-03--사이드바-알림-점-트리거-규칙) 참고.

## 경기 종료 · 결과 팝업

밸런스장이 **결과 입력**으로 승리 팀을 확정하는 순간, 해당 세션에 연결된 대상자에게 **즉시 1회** 결과 팝업(모달/토스트)을 노출한다. 별도의 "매치 결과 카드"를 본문에 두지 않는다.

| 대상 | 승부예측 활성 세션 | 승부예측 비활성 세션 |
|------|--------------------|----------------------|
| 비출전 예측 참여자 | **승자 팀** + **내 예측 적중 여부** + **배당받은 클랜코인** (적중 0 시: 실패 메시지 + 잃은 베팅 코인) | — (알림 없음) |
| 출전자 (블루/레드) | 승자 팀 확정 표시 (경기 UI 상단 배너 전환) | 승자 팀 확정 표시 (경기 UI 상단 배너 전환) |
| 관전 중인 비참가·비예측 구성원 | 승자 팀만 토스트 1회 | 승자 팀만 토스트 1회 |

**규칙**

- 팝업은 **세션별 1회**. 이미 결과를 본 사용자에게는 재노출하지 않는다(`session_results.viewed_at` 등으로 판정).
- 예측이 무효(동점·무효·재경기)인 경우 팝업 본문은 **"이번 경기는 무효 처리되어 예측이 정산되지 않습니다"** 로 대체하고 배당은 원복한다(D-BALANCE 계열 후속 결정에서 서버 정산 정책과 함께 닫는다).
- **사후 정정**(오입력·누락)은 이 팝업에 반영하지 않고 **클랜 관리 → 내전 히스토리(예정 탭)** 에서 별도 처리한다.
- 팝업이 닫히면 상태는 끝이며, 사이드바 알림 점·대시보드·통계 어디에도 "결과 미확인" 표식을 남기지 않는다.
