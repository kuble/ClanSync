## ⚠️ 검토 필요

### 1. 배틀넷 API (오버워치 게임 인증)
- **Blizzard Battle.net OAuth API** 사용 예정
- Blizzard Developer Portal 가입 및 앱 등록 필요
- 허용 범위: 계정 인증(BattleTag 조회)까지만 → 게임 내 전적 조회는 별도 약관 확인 필요
- 공식 API 외 비공식 크롤링은 ToS 위반 가능성 있음
- 참고: https://develop.battle.net/documentation

### 2. OCR 기능 (PRO)
- 게임 화면 캡처 후 닉네임 인식
- 게임사 ToS에서 화면 캡처 자동화 프로그램 허용 여부 확인 필요
- 오버워치: Blizzard ToS §2 — 자동화 도구 제한 조항 검토 필요
- **권고**: 사용자가 직접 캡처 후 업로드하는 방식으로 구현 (자동 캡처 X)

### 3. 수익 구조 (구독 서비스)
- 클랜 단위 월정액 구독 → 전자상거래법 적용
- 환불 정책 명시 필수
- 해외 사용자 대상 시 국가별 세금 처리 (VAT 등) 검토 필요

### 4. 클랜 코인 상세 설계
- 현금 구매 기능 없음 → 문화체육관광부 게임물 등급 분류 불필요
- 단, 코인으로 구매한 아이템의 현금 거래가 외부에서 발생하지 않도록 이용약관 명시 필요
- 승부예측 + 코인 조합 → 사행성 판단 기준 법률 검토 권고
  (코인이 현금 전환 불가 + 무료 획득 가능 → 통상 사행성 해당 없음, 단 법률 자문 권고)
- **프로필·네임플레이트 꾸미기**: 사용자 업로드 이미지를 허용하지 않고 **서비스 제공 에셋만** 판매·장착하는 경우, UGC 저작권·불쾌 이미지 신고 부담이 상대적으로 낮음(이용약관에도 명시 권고).

### 5. 개인정보 처리
- 닉네임, 이메일, 게임 ID 수집 → 개인정보 처리방침 필수
- 만 14세 미만 가입 제한 조항 필요
- 배틀태그 등 제3자 플랫폼 정보 저장 시 해당 플랫폼 ToS 준수

### 6. 스크림 평판 시스템
- 특정 클랜에 대한 부정적 평가 기록 → 명예훼손 리스크 최소화 설계 필요
- 클랜장만 열람 가능 구조는 리스크 낮춤
- 허위 평가 방지 로직 필요 (스크림 완료 확인 후에만 평가 가능)
