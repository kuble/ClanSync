# ClanSync PRD (Product Requirements Document)

> 멀티 게임 클랜 관리 웹서비스 — 내전·이벤트 **팀 승** 기록·참여 아카이빙, 스크림 **매칭**(승패 통계 비지원) 플랫폼
> 초기 타겟: 오버워치 → 점진적 게임 확장

---

## 개발 원칙

- 토큰 최소 사용 최적화 (문서 우선, 컨텍스트 명확화)
- 코드 최적화 필수
- 보안 최우선 (RLS, 인증, 입력 검증)
- 법적 검토 필수 (게임사 API ToS, 수익 구조)
- 나노 단위 Git 커밋/푸시
- 확장성 있는 설계 (게임 추가, 기능 확장 대비)
- 개발 중 UI 실시간 확인
- PDCA 사이클 준수

## 권한 체계

```
클랜장 > 운영진 > 구성원
```

- **클랜장**: 모든 권한, 클랜 설정, 구독 관리
- **운영진**: 밸런스메이커, 클랜 통계, 클랜 관리, 이벤트 등록
- **구성원**: 열람, 승부예측, 클랜코인 활동

## 클랜 코인 시스템

순수 가상 재화. **현금 거래 절대 불가**.

### 지급 방식
- **클랜 풀**: 기본 지급 + Premium 구독 보너스 + 구성원 기부
- **개인 풀**: 내전 참여(경기당 N코인) + 스크림 참여 보너스 + 이전 시즌 칭호 보너스

### 소비처
| 클랜 풀 | 개인 풀 |
|---------|---------|
| 홍보글 상위 노출 | 승부예측 베팅 |
| 홍보글 꾸미기 | 클랜 풀 기부 |
| 클랜 페이지 꾸미기 아이템 | 개인 프로필 꾸미기 아이템 |

> ⚠️ 논의 필요: 운영진 부정 코인 세탁 방지 방안, 코인 수치 구체화

## 점수 시스템

- **수동 점수**: 운영진이 직접 입력, 포지션별, -5.0 ~ 5.0 (실수)
- **자동 점수**: 통계 기반 자동 산출, 포지션별, -5.0 ~ 5.0 (실수)

## 핵심 기능 모듈

### 6.1 밸런스메이커 [운영진+]
- **집계일(경기일 표시)**: **KST** 기준 해당 날짜 **06:00 ~ 익일 06:00** 을 하나의 «당일»로 본다. UI에 찍히는 날짜·일간 아카이브 묶음은 **서버가 `played_at` 등으로부터 도출**해 내려주고, 클라이언트는 **자정 기준으로 임의 계산하지 않는다**.
- **세션 플로우**: **밸런스장** 1명(세션을 연 운영진)만 **① 배치 편집**; 그 외 운영진은 구성원과 동일 화면. 배치 중 보드는 **실시간 동기화**. **배치 완료** 후 맵/영웅 밴이 있으면 **웹에서 밴픽**(밸런스장·참가자 리디렉션); 비참가자는 **실시간 밴픽 관전** 가능.
- **경기 시작**: 해당 시나리오의 **밴 종료 직후**(밴 없으면 **배치 완료 직후**). **승부예측 마감 타이머**는 이 시점에 시작.
- **디스코드**: 연동 시 알림·URL은 **선택 보조**; 본 플로우는 웹 세션.
- **예측 무효**: 동점·무효·재경기 판정 시 해당 경기 승부예측 무효.
- **사후 정정**: 오입력 등은 **클랜 관리 · 내전 히스토리(예정)** 에서 처리(§6.4).
- 경기 출전 선수 선택 및 포지션 배치
- 수동/자동 점수 표시, 승률 예측 UI
- 특이사항 태그 자동 생성
- Premium: 팀 자동 밸런싱, OCR 배치, 맵/영웅 밴픽, 디스코드 알림(선택)
- Premium: 승부예측 (파리뮤추엘 방식)

### 6.2 클랜 통계 [탭별 열람 구분]
> **공통**: **내전**은 블루/레드 **팀 승**(클랜 대외 승패 아님). **스크림**은 **매칭만**·승패 통계 비지원. 상세: @docs/01-plan/clan-stats-plan.md

- **구성원 열람**: 요약 KPI, **맵·클랜 활동** 등 **클랜 단위** 지표(경기 수·유형·맵 분포·B/R 누적 등). 개인 서열은 이 범위에 넣지 않음.
- **클랜장·운영진 전용**: **경기 기록** 탭 — 일자별 **B vs R 경기 카드**(로스터·왕관) + **내전 기준 개인 승률 순위**(승차·연승·승무패·승률 등). 열람자가 제한되므로 민감 지표 허용. **§6.4**는 구성원 **개별 상세**·메달·장기 전적과 역할 분리.
- 경기 수·유형별 건수, 맵·모드별 진행 횟수, 클랜 단위 추세
- 최근 N경기 트렌드 (클랜 전체 관점)
- 출석 패턴 히트맵 (노출 범위 별도)
- 시너지 매트릭스 (운영진 위주)
> ⚠️ 개인 랭킹·상세 로스터는 **경기 기록 탭(운영진+)** 에만 두고, 구성원에게는 동 탭 비노출.

### 6.3 클랜 이벤트 [운영진+ 작성 / 구성원 열람]
- 내전·스크림·이벤트 일정 등록
- 알림: 카카오톡 or 디스코드
- Premium: 대진표 생성기 (토너먼트/더블엘리미네이션/리그전)

### 6.4 클랜 관리 [운영진+]
- 가입 요청 승인/거절
- 구성원 테이블 (배틀태그, 가입일, 참여일, 기부금액, 권한)
- 개인 상세: **해당 구성원 기준 승·패·참여 등 개인 통계**, 포지션별 점수, 시너지, 메달(칭호) — **§6.2 클랜 통계와 구분**
- 메달 예시: 맵장인(특정 맵 10승+), 개근왕(월 참여율 1위), 탱커 맛집(탱커 승률 70%+)
- **(예정)** **내전 히스토리** 탭: 잘못 입력된 값 수정, 사후 요소 추가 등 — **§6.1** 실시간 밸런스 플로우와 별도

### 6.5 클랜 스토어
- 클랜 페이지 꾸미기 아이템
- 개인 프로필 꾸미기 아이템 (게임별 독립 적용)

### 7. MainGame Page (게임 커뮤니티)
- 클랜 홍보 게시판 [운영진+ 작성]
- 스크림 신청 게시판 + 클랜 간 채팅방 (스크림 기간 한정)
- 스크림 자동 매칭 (티어대·연령대 등 조건 반영)
- 클랜 순위표 (민감 지표 포함 여부 논의 필요)
- 스크림 평판 시스템 (클랜장만 기입/열람)

## 구현 메모 (📌)

### 플레이어 프로필 페이지
- 클랜 홍보 게시판에서 "가입 신청하기"를 누른 **신청 대기 중인 클랜 목록**을 플레이어 프로필에서 확인할 수 있어야 함
  - 신청 클랜명, 신청 날짜, 현재 상태(대기 중 / 승인 / 거절) 표시
  - 대기 중인 신청은 취소 가능
  - 관련 파일: `mockup/pages/main-game.html` → `openJoinModal()` / `submitJoin()`

### 에셋 교체 대기 목록
> 현재 플레이스홀더로 구현되어 있음. 실제 이미지/아이콘 준비되면 교체 필요.

- [ ] **티어 아이콘** — 클랜 홍보 카드 및 LFG 카드의 티어 배지를 실제 게임 랭크 이미지로 교체
  - 현재: `B / S / G / P / D / M / GM` 문자 + 색상 배지 (`.tier-ic` CSS 클래스)
  - 교체 위치: `TIER_IC` 상수 → `abbr` 대신 `<img src="...">` 적용
  - 관련 파일: `mockup/pages/main-game.html` → `TIER_IC`, `promoTagsHtml()`, `lfgTierIconsHtml()`

- [ ] **클랜 배지 아이콘** — 카드 하단 및 드로워의 배지를 실제 아이콘 이미지로 교체
  - 현재: 이모지 (⚔️ 🎙️ 🏆 💬 등)
  - 교체 위치: `BADGE_DEFS` 상수의 `icon` 필드
  - 관련 파일: `mockup/pages/main-game.html` → `BADGE_DEFS`, `clanBadgesHtml()`

- [ ] **게임 사이드바 썸네일** — 게임 선택 사이드바의 게임 커버 이미지 교체
  - 현재: 이모지 또는 텍스트 약자
  - 관련 파일: `mockup/pages/main-game.html` → 사이드바 게임 목록 HTML
