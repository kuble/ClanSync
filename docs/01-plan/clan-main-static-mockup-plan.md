# 클랜 메인 · 서브 영역 정적 목업 플랜

> **범위**: `mockup/` + `_hub.html` 만. Next.js·Supabase 연동은 제외.  
> **참고**: 제품 라우트는 @docs/01-plan/pages.md · 요구는 @docs/01-plan/PRD.md §6.x

**구현 반영(정적 목업)**: `mockup/pages/main-clan.html` + `mockup/scripts/clan-mock.js` + `_hub.html` 사이드바 **목업 권한**(sessionStorage + `?role=` 로 iframe 로드). 서브뷰·PRO·탭·모달은 플랜에 맞춤.

---

## 1. 목적

- 실제 구현 전에 **화면 구조·정보 우선순위·운영진/구성원 시야 차이**를 고정한다.
- 디자이너·기획·개발이 같은 HTML을 보며 **절차(어떤 화면 다음에 무엇이 보이는지)**를 맞춘다.
- `_hub` iframe에서 **빌드 없이** 즉시 확인 가능해야 한다.

---

## 2. IA(정보 구조) — 제품 URL과의 대응

| 제품 경로(개념) | 정적 목업에서의 이름 | PRD 모듈 |
|----------------|---------------------|----------|
| `.../clan/[clanId]` (루트) | **대시보드** | MainClan 요약 |
| `.../balance` | **밸런스메이커** | §6.1 |
| `.../stats` | **클랜 통계** | §6.2 |
| `.../events` | **클랜 이벤트** | §6.3 |
| `.../manage` | **클랜 관리** | §6.4 |

허브에서는 **파일 하나 + 화면 전환**이어도 되고, 나중에 HTML 분할도 가능하다. 현재는 `pages/main-clan.html` 내 해시/JS 전환으로 묶여 있음.

---

## 3. 권한(운영진 vs 구성원) 목업 전략

정적 HTML에는 로그인이 없으므로 다음 중 하나로 **기획 의도만** 드러낸다.

1. **권한 배지**: 화면 상단에 `[목업: 클랜장 보기]` 같은 고정 캡션.
2. **이중 블록**: 같은 섹션에 “구성원에게 보이는 요약” / “운영진 전용” 카드를 나란히 배치.
3. **선택적 (후순위)**: `?role=member` 쿼리로 CSS `display` 토글하는 스크립트.

**원칙**: 밸런스·관리는 **운영진+** 전용이므로, 구성원 시나리오를 볼 때는 **사이드바에서 해당 메뉴 비활성·잠금 안내**를 넣을지 PRD와 함께 결정한다.

---

## 4. 서브 페이지별 목업에 넣을 블록 (플래닝 체크리스트)

### 4.1 대시보드 (클랜 홈)

- 클랜 배너·한 줄 메타(게임명, 인원, 태그).
- 공지 영역, 배지(업적) 그리드.
- MVP·다가오는 일정·내 참여율 등 **요약 카드** (이미 상당 부분 존재).
- 커뮤니티(MainGame) 이동 진입점.
- **다음 검토**: “최근 경기 1~2건” 카드 추가 여부.

### 4.2 밸런스메이커

- 경기 단위 탭 또는 목록(작성중 / 종료 예시).
- 맵·모드(제어/호위 등) 메타.
- A/B 팀 로스터: 닉·태그·역할(탱/딜/힐)·수동·자동 점수.
- 승률 예측 바(목업 수치) + **PRO** 배지가 붙는 항목(자동 밸런스, OCR, 승부예측) 구분.
- 특이사항 태그 영역(자동 생성 **표시만**).
- **다음 검토**: 맵/영웅 밴픽 PRO 존을 빈 슬롯으로라도 배치할지.

### 4.3 클랜 통계

- **재기획 중**. 집계 단위는 **클랜**(경기 아카이브·클랜 승률·클랜 경기 기준 순위표). 개인 대시보드처럼 보이는 블록은 지양.
- 상세 블록·순서·MVP: **@docs/01-plan/clan-stats-plan.md** 가 단일 기준.
- 정적 목업 `view-stats`: 현재 **백지(준비 중)** — 플랜 확정 후 HTML을 블록 단위로 맞춤.

### 4.4 클랜 이벤트

- 목록 뷰: 일정 카드(날짜·시간·장소·유형 배지).
- 캘린더 뷰: 월 그리드 + 일정 있는 날 표시(현재 수준 유지·다듬기).
- 운영진: `일정 등록` 버튼 위치·모달(빈 폼 목업 가능).
- **다음 검토**: PRO 대진표 생성 진입점을 버튼만 두고 비활성화할지.

### 4.5 클랜 관리

- 가입 요청 테이블 + 승인/거절 버튼(alert 목업 유지 가능).
- 구성원 테이블: 닉네임, 게임 ID, 역할, 가입일 — **PRD의 참여일·기부금액** 컬럼 추가 여부 결정.
- 멤버 상세(드로어/별도 블록): 포지션 점수·시너지·메달 — **1명 분 샘플**만 넣어도 됨.
- 클랜 프로필(태그·연령·링크) 요약 — 이미 일부 존재, PRD 필드와 항목 맞추기.
- **다음 검토**: 클랜장만 구독/결제 UI를 이 페이지에 둘지, 별도 목업 페이지로 둘지.

### 4.6 클랜 스토어

- 클랜 풀 / 개인 풀 잔액 표시.
- 탭: 클랜 꾸미기 vs 개인 꾸미기.
- 아이템 카드: 이름, 설명, 가격(코인), 구매 버튼, **PRO 잠금** 예시 1건 이상.
- **다음 검토**: 코인 숫자는 고정 목업으로 둘지, 표기 규칙(천 단위 콤마 등)만 문서화할지.

## 5. 구현 순서 제안 (정적 목업만)

1. **P0 — 내비게이션 일관성**: 사이드바·해시·활성 상태·허브 안내 문구 유지.
2. **P1 — 대시보드**: PRD와 안 맞는 카드/카피 정리, 빈 상태(데이터 없을 때) 1케이스.
3. **P2 — 밸런스 / 통계**: PRO 경계선을 시각적으로 통일(배지·disabled 스타일).
4. **P3 — 이벤트 / 스토어**: 캘린더·상품 그리드 밀도·모바일 줄바꿈.
5. **P4 — 관리**: 멤버 상세·추가 컬럼·권한별 시나리오.

## 8. 미결 정책 (기획에서 먼저 결정하면 목업이 쉬워짐)

- 구성원에게 **통계·이벤트**까지 동일 네비를 보여줄지, 메뉴 자체를 숨길지.
- 클랜 순위표·평판(MainGame)과 **클랜 메인** 사이 카피/진입 관계.
- PRO 기능을 목업에서 **항상 보이게(잠금)** 할지, 아예 숨길지.

이 문서는 허브 목업 작업의 **스코프·우선순위·PRD 매핑**을 고정하는 용도이며, 합의가 바뀌면 여기와 `main-clan.html`을 함께 갱신한다.
