# 용어 사전 (Glossary)

이 문서들에 자주 나오는 웹 용어를 한 줄로 풀어쓴 작은 사전. 페이지 문서 본문에서는 처음 나올 때만 `(= 라우트 가드)` 식으로 한 번 병기하고, 그 다음부터는 한국어로만 쓴다.

## 주소·이동 관련

| 한국어 표기 | 영문/기술 표현 | 한 줄 설명 |
|-------------|----------------|-----------|
| 주소 | URL · 라우트(Route) | 웹 화면의 위치. 게임의 "맵 ID"와 비슷하게 화면 ID 역할 |
| 화면 진입 잠금 | 라우트 가드(Route Guard) | 조건 미충족이면 다른 화면으로 자동 이동 |
| 자동 이동 | 리다이렉트(Redirect) | "당신은 못 들어와요, 여기로 가세요" 식으로 보냄 |
| 주소 옵션 값 | 쿼리 파라미터(Query Parameter) | 주소 끝에 `?key=value`로 붙는 옵션 |
| 주소 안 ID | 패스 파라미터(Path Parameter) | 주소 안에 `/clan/[clanId]`처럼 박힌 ID |
| 화면 안 작은 화면 | 탭(Tab) · 서브뷰(Sub-view) | 같은 주소 안에서 위쪽/사이드 메뉴로 갈아끼우는 작은 화면 |

## UI·인터랙션 관련

| 한국어 표기 | 영문/기술 표현 | 한 줄 설명 |
|-------------|----------------|-----------|
| 팝업 창 | 모달(Modal) | 화면 가운데 떠서 다른 조작을 막는 창 |
| 옆 패널 | 드로어(Drawer) | 화면 옆에서 미끄러져 나오는 패널 |
| 알림 띠 | 토스트(Toast) | 잠깐 떴다 사라지는 안내 띠 |
| 풍선 도움말 | 툴팁(Tooltip) | `?` 아이콘 등에 마우스를 올리면 뜨는 작은 설명 |
| 입력 검사 | 폼 검증(Form Validation) | 입력값이 규칙(이메일 형식, 글자 수)에 맞는지 검사 |
| 항상 보이는 좌측 메뉴 | 사이드바(Sidebar) | 게임의 좌측 메뉴와 동일 |
| 위쪽 메뉴 | 헤더 / 내비바(Navbar) | 화면 맨 위 가로 메뉴 |
| 빈 상태 화면 | Empty State | 데이터가 0건일 때 보여주는 안내 화면 |

## 로그인·세션 관련

| 한국어 표기 | 영문/기술 표현 | 한 줄 설명 |
|-------------|----------------|-----------|
| 출입증 | 세션(Session) · 토큰(Token) | 로그인 상태를 들고 다니는 신분증 |
| 다른 서비스로 로그인 | OAuth | 디스코드/배틀넷 같은 외부 계정으로 로그인 |
| 권한 | Role · Permission | 운영자/일반 회원 등 할 수 있는 일의 범위 |
| 구독 등급 | Plan · Tier | 기본(Free) / 유료(Premium) 구분. 게임의 패스 등급 비슷 |

## 저장·데이터 관련

| 한국어 표기 | 영문/기술 표현 | 한 줄 설명 |
|-------------|----------------|-----------|
| 브라우저 영구 저장소 | localStorage | 탭을 닫아도 남는 작은 저장 공간 |
| 브라우저 임시 저장소 | sessionStorage | 탭을 닫으면 사라지는 저장 공간 |
| 서버 데이터 모양 | 스키마(Schema) · 데이터 shape | 어떤 필드가 어떤 타입으로 들어 있는지의 규격 |
| 데이터 한 줄 | 레코드(Record) · 로우(Row) | 표의 한 줄 |
| 검문소 | 미들웨어(Middleware) | 서버에서 화면 코드에 닿기 전에 한 번 거치는 단계. 로그인/권한 검사에 자주 씀 |

## 게임 비유 빠른 참고

| 게임 개념 | 웹 대응 |
|-----------|---------|
| 길드장 / 임원 / 일반 길드원 | 클랜장(leader) / 운영진(officer) / 구성원(member) |
| 길드 입장 자격 검사 | 라우트 가드 (로그인 → 게임 인증 → 클랜 소속 → 역할) |
| 인벤토리 슬롯 | 뱃지 케이스 슬롯, 네임플레이트 슬롯 |
| 패스 등급에 따른 보상 잠금 | 구독 등급별 기능 잠금 (`mock-hide-on-free`) |
| 인스턴스 던전 입장 조건 | 화면 진입 잠금 |
| 게임 클라이언트 ↔ 서버 패킷 | 웹의 API 요청/응답 |

## 사용 규칙

1. 페이지 문서 본문에서 위 용어가 처음 나올 때만 `한국어 (= 영문)` 식으로 한 번 병기.
2. "구현 참고 (개발자용)" 섹션 안에서는 영문/기술 용어만 써도 OK.
3. 새 용어가 페이지 문서에 등장하면 이 사전에 추가한 뒤 본문에서 사용한다.
